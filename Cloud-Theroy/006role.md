# 보안과 권한 관리
## AWS 계정 루트 사용자
모든 서비스에 대한 전체 액세스 권한
단일 계정 모델에서는 권한을 제한할 수 없음
일상적인 상호작용에 사용하면 안됨
- 최소 권한의 원칙

## AWS Identity and Access Management(IAM)
- 사용자, 그룹 및 역할 생성과 관리
- AWS 서비스와 리소스 액세스 관리
- 액세스 제어 분석

## 보안 주체
- AWS 리소스에 대한 작업 및 운영을 요청할 수 있음
- 사용자, 애플리케이션, 페더레이션 사용자 또는 수임된 역할 일 수 있음

## IAM 사용자
- IAM 사용자는 사람인 사용자 또는 워크로드를 나타내는 AWS 계정 내의 엔터티임
- 각 사용자는 고유한 자격 증명을 소유함
- IAM 사용자는 권한에 따라 특정 AWS 작업을 수행할 수 있음

## IAM 사용자 그룹
- IAM 사용자를 IAM 사용자 그룹에 할당함
- IAM 사용자 그룹에 정책을 연결하면 해당 그룹 내의 모든 사용자에게 정책이 적용됨

## IAM 역할
- 임시 자격 증명을 사용하여 특정 사용자 또는 서비스에 일련의 권한을 위임함
- 사용자는 다른 사용자와 자격 증명을 공유하지 않고 역할을 수임함
- 권한은 수임한 역할로 작업을 수행하는 동안에만 유효함

## 역할 수임
1. api 호출을 사용해 역할을 수임
2. 임시 보안 자격 증명 반환
3. 임시 보안 자격 증명 사용

## 보안 정책 범주
- 정책유형
    1. 최대 권한 설정
        - IAM 권한 경계
        - AWS Organizations 서비스 제어 정책
    2. 권한 부여 
        - IAM 자격 증명 기반 정책
        - IAM 리소스 기반 정책

### 권한 부여
- 자격증명 기반 정책은 사용자, 그룹, 역할에 할당됨
- 리소스 기반 정책은 리소스에 할당함
