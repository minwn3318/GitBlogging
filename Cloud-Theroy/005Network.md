## 클래스 레스
Classless Inter-Domain Routing CIDR

블록이라는 IP주소 범위를 지정

10.22.0.0/16

. : 네트워크 또는 서브넷을 지정하는 점 표기법
/ : 서브넷 마스크에 대한 슬래시 표기법

/00 를 사용하면 총 32개의 비트 중 00개를 네트워크 용으로 쓰는 걸 가리킴 

## aws VPC 관련주제
### Amazon VPC
- 워크로드를 논리적으로 격리 할 수 있음
- 리로스에 사용자 정의 액세스 제어 및 보안 설정 사용 가능
- 단일 AWS 리전에 연결됨 - 여러 가용 영역에 할당함
- 기본 VPC는 이미 생성됨

### 서브넷
- VPC CIDR 블록의 하위 집합
- 서브넷 CIDR 블록은 중첩 도리수 없ㄷ음
- 각 서브넷은 하나의 가용영역 내에 포함됨
- 하나의 가용 영역에 여러 서브넷을 포함할 수 있음
- 주소 5개 예약됨

### 퍼블릭 서브넷
- 퍼블릿 서브넷은 인바운드 및 아웃바운드 인터넷 트래픽을 사용하는 리소스가 포함됨
- 퍼블릭 서브넷에 필요한 항목

라우팅 테이블
- VPC가 네트워크 트래픽을 라우팅하는데 사용하는 규칙 집합
- 인터넷에 대한 경로 필요

인터넷 게이트웨이
- VPC의 리소스와 인터넷 간의 통신허용 

퍼블릭 IP 주소
- 인터넷에서 연결할 수 있는 IP주소
- 네트워크에서만 연결할 수 있는 프라이빗 IP 주소 보호

### 인터넷 게이트 웨이
- VPC 간의 인스턴스와 인터넷 간의 통신을 허용함
- 인터넷으로 라우팅 가능한 트래픽에 대한서브넷 라우팅 테이블에서 대상을 제공함
- NAT를 수행하여 네트워크이 IP주소를 보호함

### 라우팅 테이블
- VPC는 암시적 라우터가 있음
- 라우팅 테이블을 사용하여 네트워크 트래픽 경로를 제어할 수 있음
**EIP**는 유료임

### NAT 게이트웨이 (유료임)
- 프라이빗 서브넷이 외부와의 통신하기 위한 직접 개통처

### 네트워크 ACL
- 서브넷 경계에서 방화벽 역할을 함
- 기본적으로는 모든 인바운드 및 아웃바운드 트래픽이 허용됨
- 상태 비저장이므로 모든 트래픽에 대한 명시적인 규칙이 필요
- 번호가 낮은 규칙부터 시작해 규칙을 차례로 평가함

ip 프로토콜 포트 또느 IP 주소를 기준으로 트래픽을 허용함
