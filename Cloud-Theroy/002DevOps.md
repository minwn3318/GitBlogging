# 애자일, DevOps, CICD, 클라우드 네이티브
## 클라우드 네이티브
- 클라우드의 장점을 최대한 활용할 수 있도록 애플리케이션을 개발하고 운영하는 방법론
<br>

## 데브옵스
- 개발과 운영자 영어를 합친 것
소프트웨어를 더 빠르고 안정적으로 개발, 배표, 운영하기 위한 조직 문화와 방법론
<br>

## CI
개발자가 작은 단위로 코드를 통합하고 자동으로 빌드와 테스트하는 프로세스
code - build-test
변경사항을 주기적으로 빈번하게 머지 해야함
<br>

## CD
코드 변경 사항을 자동으로 준비하고 필요시 자동배포까지 이어지는 과정임
release-deploy-monitor
<br>

## 개발론
### 폭포수 모델
- 장점
단순한 모델
정형화된 단계별 접근방법
명확한 진행상황 파악

- 단점
완벽을 요구하는 요구사항 작성
어려운 변경 요청 사항
동작을 프로세스 후반에서 확인가능
일정 지연 가능성
<br>

### 애자일 개발론
- 개발이 완료 된 후 배포
기존의 복잡하고 무거운 개발 방법론에서 벗어나 가볍고 유연한 개발론으로 탄생
변화에 유연하게 대응하여 고객 만족도를 높이기 위해 반복적인 개발, 지속적인 피드백을 강조


## 서비스 아키텍처 형태 IaaS PaaS SaaS

### On Premise
- 회사가 인프라, 미들웨어, 어플까지 다 관할
<br>

### IaaS
- 인프라만 관할
<br>

### Paas
- 인프라, 미들웨어까지 관할
<br>

### SaaS(간편하긴 한데, 커스텀 마이너징하기 힘듦)
- 어플리케이션까지 관할
