# 클라우드 
<br>

## 클라우드 사용이유
1. 민첩성
- 출시 시간 단축
- 더욱 광범위한 혁신 추진
- 필요에 따른 크기 조정 - Elasfic/scale
<br>

2. 복잡한 및 위험
- 비용 최적화
- 보안 취약서 최소화
- 복잡한 관리 작업 감소
<br>

3. 가용영역
- 리전 내의 데이터 센터
- 내결함성을 갖도록 설계됨
- 고속 프라이빗 링크를 사용하여 상호 연결됨
- 고가용성 구성에 사용됨
<br>

## AWS 클라우드 서비스 
<br>

### 리전
리전은 3개 이상의 가용영역(논리적 데이터 센터)
- 완전히 독립적
- AWS 네트워크 인프라 사용
- 여러 개 가용 영역 포함
<br>

### 가용영역
가용영역은 1개 이상의 데이터센터(논리적 데이터 내의 물리적 위치)
<br>

> **집합 정도**
> 리전 > 가용영역
<br>

### local zones(엔지니어 입장)
- 고객과 더 가까운 물리적 위치에 두는 인프라 옵션
- 출장소
<br>

### 엣지로케이션(고객 입장)
- 오리진으로부터 미리 데이터를 가져와 고객에게 가까이 전달해주는거
<br>

## AWS 클라우드 설계 시 주의점
<br>

1. spof
single pointer of failure (단일 진입점 실패)
- 실무에 많이 사용
- 이것때문에 이중화가 기본
<br>

2. 리전 선택
- 거번너스 (디지털 정책)
- 지연시간 (레이턴시)
- 서비스 가용성
- 비용
<br>
