# 데이터베이스 서비스
## 데이터베이스 형태

### 관계형 데이터베이스
- 행과 열로 이뤄진 테이블

### 비관계형 데이터베이스
- 키 값, 와이드 컬럼, 그래프, 문서 또는 기타 모델

## 아마존 RDS
- 하드웨어 운영체제 및 데이터베이스 소프트웨어 배포 및 유지 관리
- 모니터링 기본 제공

### 제공 기능
1. 저장 시 및 전송 중 데이터 암호화
    - 업계 규정 준수
2. 자동 다중 AZ 데이터 복제
3. 컴퓨팅 및 스토리지 크기 조정
4. 애플리케이션 가동 중단 최소화

### 용어 설명
1. 다중 AZ 배포
- 다중 AZ 배포
다른 가용영역의 대기 디비 인스턴스에 데이터 복제 읽기전용 목적으로 사용되지 않음

2. 읽기 전용 복제본
- 읽기 전용 복제본을 사용하여 다음을 수행할  수 있음
- 읽기 중심의 워크로드 처리를 위해 수평 크기 조정
- 오프로드 리포팅
- AWS 리전 간 복제

## aurora db 클러스터
- 클러스터는 하나이상의 디비 인스턴스와 클러스터 볼륨으로 구성됨
- 프라이머리 인스턴스는 읽기 쓰기 작업을 수행
- aurora 복제본은 읽기 전용임
- 클러스터 볼륨은 여러 가용영ㄱ에 거렻이쓴 가상 데이터베이스 스토리지 볼륨임

## dynamo db
완전 관리형 노에스큐엘 데이터베이스 서비스
- 규모에 따른 일관된 성능 제공
- 서버 관리 불필요
- 엔터 프라이즈 지원
- vpc 외부에 만들어짐 (s3도 외부에 만들어짐)

### dynamo db 일관성 옵션
다이나모디비는 일반적으로 1초 내에 한 리전 내 3개의 가용영역에 걸쳐 테이블 데이터를 복제함
- 최종적 읽기 일관성 (최종적으로 일관성 있으면 됨)
- 강력한 읽기 일관성

### 캐시해야하는 항목
- 쿼리속도가 느리고 비용이 많이드는 데이터
- 자주 엑세스하는 데이터
- 비교적 정적 상태로 유지되는 정보

### 캐싱 아키텍처
- 자주 사용하는 데이터를 보관하는 임시 장소
- 디비에 매번 접근하지 않고 빠르게 가져올 수 있음

### 캐싱 전략
1. 애플리케이션이 캐시에 데이터 요청
2. 캐시 미스
3. 애플리케이션이 데이터베이스에서 누락된 데이터 요청
4. 데이터베이스에서 데이터 반환
5. 애플리케이션이 반환된 값을 캐시에 작성

### 캐싱 전략 - write-through
1. 애플리케이션이 데이터베이스에 데이터를 기록
2. 애플리케이션이 캐시에도 데이터를 기록
