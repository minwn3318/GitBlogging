# spring boot
- 단독 실행 가능한 스프링 애플리케이션을 생성한다
- 내장형 톰켓, 언더토우를 내장하고 있으며 war 파일을 이용해 배포 할 필요가 없다
- 기본으로 설정되어 있는 starter 컴포넌트들을 쉽게 추가할 수 있음
- 자동 구성을 통해 설정이나 3rd-party 라이브러리를 구성할 수 있다
- 상요화에 필요한 통계, 상태 점검 및 외부 설정을 제공한다
- 설정을 위한 xml 코드를 생성하거나 요구하지 않는다

# Starter
스프링 부트에서 스타터 개념을 도입해 기발자들이 프로젝트에 필요한 의존성을 빠르고 편리하게 관리할 수 있게 해준다
- 편의성
- 일관성
- 자동 구성

빌드 도구
- maven
xml을 사용함(pom.xml)
규칙 기반 설정으로 초기 설정이 간단하고 관습적임
중앙 maven repository를 통해 라이브러리를 쉽게 가져올 수 있다

-gradle
grovvy나 kotlin dsl을 사용
우연하고 강력한 dsl을 통해 복잡한 빌드 로직을 쉽게 표현할 수 있음

@springbootapplication
- 생성된 애플리케이션 클래스는 메인 메소드가 존재하며 애플리케이션 런을 호룿해 실행시킴
항상 프로젝트이 최상단 위치해야한다

# 디자인 패턴
- 특정 문맥에서 공통적으로 발생할 수 있는 문제에 대해 재사용 가능한 해결책

mvc(model-view-controller)
애플리케이션을 3가지 역할로 구분하여 개발하는 방법론
비즈니스 로지고가 ui 로직을 분리하여 코드 종속성을 줄이고, 구성 요소간 역할을 명확하게 함으로써 분리가 ㅜ십고 협업을 용이하게 한다
view(interface)->controller(api)->model

controller
- 사용자 입력을 받아 데이터를 변경하거나 이를 뷰에 전달하는 역할
- 컨트롤러는 모델과 뷰 사이에서 데이터를 주고 받으며 전체적인 흐름을 조절하느 역할을 한다
- 스프리의 겨우 @controller 어노테이션을 통해 컨트롤러 역할을 하는 스프링 빈 생성을 할 수 있따

@GetMapping :tttp에서 자원을 지정했을 때 거기에 해당하는 메소드를 실행

model : 데이터를 처리하는 영역이다
데이터 저장소와 연동하여 사요자가 입력한 데이터나 사용자에게 출력할 데이터를 다룬다
본 예제에서는 다순히 userName만 활용한다

view
어플리케이션이 처리한 데이터나 그 작업 결과를 사용자에게 출력할 화면으로 만든다
모델 컨트롤러와 종속성이 없도록 구현해야한다

spring mvc
- 스프링에서 제공하는 웹 모듈
- model, view, controller 세가지 구성 요소를 사용하여 사용자의 다양한 http request를 처리하고 단순한 텍스트 형식의 응받부터 레스트 형식의 응답은 물론 view를 표시하는 html을 리런타흔 응답까지 다양한 응답을 지원하는 프레임워크이다
프론트 컨트롤러 패턴에 기초한 웹 mvc 프레임워크이다
모든 사용자 요청은 dispatcherservlet은 프론트 컨트롤러 패턴에 의해 설계된 진입점으로 사용자 용청을 분석하고 해당 요청을 처리할 컨트롤러를 찾아 호출한다

정리
컨트롤러는 클라이언트의 요청을 받는다
리넡 값으로 스트리 -> 머스테치를 반환한다
컨트롤러에서 username이라는 변수를 등록하고 값을 연결해 뷰 템플릿에서도 사용할 수 있도록 한다

타임리프 <- 이친구도 템플릿 의존성임 확장작는 html
템플릿 안에 html이 들어가도 괜춘해짐
