# LLM과 Langchain 코드 비교
## 개요
AI 에이전트 구축 미니프로젝트를 하며 다른 사람의 코드리뷰를 하게 되었다. 

모델 호출 시, 나는 chat.completaion.create()을 사용하였고, 그 사람의 코드에는 invoke()를 사용하였다.

결과론적으로 보았을 때는 차이가 없었다. 

둘 다 똑같이 모델 선언 후, 프롬프트를 짜고, 모델을 호출하여 응답을 얻은 것이기 때문이다.

즉, 같은 행위를 한 것이이게 겉보기에는 굳이 두 개로 분리할 이유가 없었다.

나는 이것이 궁금해져서 이 두 방식의 차이점을 여기에 정리하게 되었다 

---
## LLM? LangChain?
### LLM
- 대규모 언어모델로 프로그래밍 시 많이 사용하는 모델은 OpneAI가 있다

### LangChain?
- open ai와 같은 생성형 언어모델들을 활용한 애플리케이션 생성을 단순화 하도록 한 것이다. 즉, **프레임 워크다**

따라서, LLM이 **원본 그 자체의 소스**라고 하면, Langchain은 원본 소스를 조금 **더 편리하게 사용할 수 있도록 한 도구**이다.
그러니 이 둘의 관계는 LLM -> LangChain으로 **LLM이 더 Low한 단계**, **LangChain은 High한 단계이다**


---
## LLM과 LangChain 단계에 따른 차이

서로 다른 단계에 있으면 당연히 차이가 난다. 그럼 어떤 차이가 있느냐,

- LLM의 경우, 더 시스템과 가까운 단계이니 세밀한 조정이 가능하다

- LangChain의 경우, 시스템과 먼 관계이다. 이말은 유저와 가깝다는 뜻으로 유저에게 간편하게 더 단순화된 API를 제공해준다
