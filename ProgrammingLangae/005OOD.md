# 객체지향 기초
## 객체의 개념
- 현실 세계의 객체를 필드와 메서드로 모델링 한 것
- 상태를 필드로 정희하고 동작을 메서드로 정의

### 절차지향 프로그래밍
- 일련의 동작을 순서에 맞추어 단계적으로 실행하도록 명령어를 나열
- 데이터를 정의하는 방법보다는 명령어의 순서와 흐름에 중점
- 수행할 작업을 예상할 수 있어 직관적인데, 규모가 작을 때는 프로그래밍과 이해하기가 용이

## 객체지향 프로그래밍
- 소프트웨어의 규모가 커지면서 동작과 분리되어 전 과정에서 서로 복잡하게 얽혀 있는 데이터를 사용
- 절차 지향 프로그램은 추후 변경하거나 확장하기도 어려움
- 현실 세계를 객체 단위로 프로그래밍하며 객체는 필드와 메서드 코드르 하나로 묶어 표현

### 캡슐화
- 데이터를 정의하는 방법보다는 명령의 순서와 흐름에 중점
데이터는 숨기고 메서드로만 외부와 통신

### 상속
- 상위 객체를 상속받은 하위 객체가 상위 객체의 메섣와 필드를 사용하는 것
- 변경에는 닫혀있고(지양), 확장에는 열려있다(지향지향지향)
- 새로운걸 만들 때 기본적 내용을 변경하지 않고 추가해서 만들면 되늰 거임

## 다형성
- 대입되는 객체에 따라서 메서드를 다르게 동작하도록 구현하는 기술

## 추상화
- 현실세계의 객체는 수많은 상태가 있고 다양한 동작을 하지만, 클래스에 모두 포함하기에는 어렵기에 추상화하는 과정이 필요
- 추상화는 현실세계의 객체에서 불필요한 속성을 제거하고 중요한 정보만 클래스로 표현하는 일종의 모델링 기법

객체 생성과 참조 변수
- 클래스 이름 변수
변수 = new 클래스 이름()

기초타입과 참조타입
값 복사
보통 변수를 옯길 때는 스택에 있는 값을 옮김
문제는 기본 변수는 스택에 값이 있으니 옮겨도 문제없는데, 참조변수는 스택에 주소가 있어서 얕은 복사이 이루어짐

자바에서 세터케터를 쓰는 이유
그냥 변수에 바로 접근하게 해도 되잖음
그냥 쓰면 다른 객체들이 모든거에 영향을 받고 잇음 전파를 막음

메서드 오버로딩
- 같은 이름의 메서드를 여러개 정의하는 것
- 매개변수의 개수나 타입이 달라야 함
- 반환 타입만 다른 것은 오버로딩이 아님

생성자
-생성자의 역할 : 객체를 생성하는 시점에서 필드를 다양하게 초기화

final 한번 설정하면 변경안할 거임
- final은 보통 생성할 때만 할당되면 추천됨 => 이후 무언가 변경 메서드가 있으면 추천안해줌

this는 그냥 매겨변수
this() 생성자, 메서드

연속호출
반환값이 void라면 발생
연속호출 원리
스택에 객체가 있다 가정하자
힙에는 객체의 값들이 할당되어 있다

스택의 객체에게 힙에 있는걸 호출하게 한다 이때 이 리턴은 이 객체 자체를

인스턴스 멤버와 정적 멤버
- 자바의 static 키워드는 클래스 차원에서 공유되는 변수를 정의할 때 사용함
- 인스턴스 변수 : 각 객체마다 별도로 존재하는 변수
- 정적 변수 : static으로 선언되어 모든 인스턴스가 공유하는 하나의 변수

인스턴스 변수는 객체별로 관리 : 객체를 생성할 때 인스턴스 변수도 객체가 소멸될 때는 자동으로 소멸
정적 변수 혹은 클래스 변수 : static 키워드로 지정하여 모든 인스턴스가 공유하는 필드
반면 정적 변수는 클래스 로더가 클래스를 메서드 영역에서 적재할 때 생성
- 객체와 관련된 인스턴스 변수를 사용할 수 없다
- 객체 자신을 가리키는 디스 키워드를 사용할 수 없다

상속
필요성
- 핵심 구현체로 세부 구현체를 만들 때
- 세부구현체를 만들어도 보통 기준 소스 코드인 부모를 위주로 쓰게 만들어야함
- 다중상속이 안됨

오버라이딩
규칙
- 부모 클래스이 메서드와 동일한 시그니처를 사용 심지어 반환 타입까지 동일해야함
- 부모 클래스이 매서드보다 접근 범위를 더 좁게 수정할 수 없다
- 추가적인 예외가 발생할 수 있음을 나타낼 수 없다

구체화가 부족한 것을 추상화라고 함

패키지
- 클래스 파일을 묶어서 관리하기 위한 수단으로 파일 시스템의 폴더를 이용

- 패키지마다 별도의 이름공간이 생기기 때문에 클래스 이름의 유일성보장
- 클래스를 패키지 단위로도 제어할 수 있기 때문에 좀 더 세밀하게 접근 제어

import 
패키지의 경로를 미리 컴파일러에게 알려주는 문장

final 클래스
더 이상 상속할 수 없는 클래스

final 메서드
클래스 내부의 메서드를 오버라이딩 할 수 없음
