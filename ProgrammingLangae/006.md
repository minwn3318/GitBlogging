# 타입변환과 다형성
객체들은 상속관계에서만 형변환이 가능함
이중 자식개체를 부모타입처럼 사용할 수 있음

(예시)
```
부모타입 부모변수;
자식타입 자식변수 = 자식객체;
부모변수 = 자식개체;
```
- 해당경우 자식개체는 부모타입처럼 사용됨
- 부모에 있는 함수를 호출시, 자식에도 같이 재정의된 함수가 있다면 해당 자식함수가 호출됨 (자식에도 함수가 있는데 재정의 되어 있지 않다면 부모것이 호출됨)
자동형변환 : 자식은 부모타입으로 업케스팅 가능

(예시)
```
부모타입 부모변수;
자식타입 자식변수 = 자식개체
부모변수 = 자식개체
자식타입 자식변수2 = (자식타입)부모변수
```
- 해당경우 자식변수2는 부모변수를 자식타입으로 받아들여 부모변수에 있는 자식개체를 자식타입으로 사용할 수 있음 
- 자식함수를 전부 사용 가능함

**즉, 부모변수를 기준으로 안에 자식개체가 있을 때는 일반적으로 업케스팅(부모에게서 받은 함수만 사용 가능), 이상태에서 아예 부모변수를 자식변수로 형변환 하면 다운케스팅(자식이 가진 모든 함수를 사용 가능)**

# 추상 메서드
- 메서드 본체를 완성하지 못한 메서드. 무엇을 할지는 선언할 수 있지만 어떻게 할지는 정의할 수 없음

추상 클래스
- 보통 하나 이상의 추상 메서드를 포함하지만 없을 수도 있음
- 주로 상속 계층에서 자식 멤버의 이름을 통일하기 위하여 사용

인터페이스 의미
- 자바에서 인터페이스란 클래스가 구현해야할 메서드들의 집합으로 각 객체 간의 통신을 위한 계약을 정의함

인터페이스 장점
- 인터페이스만 준수하면 통합에 신경 쓰지 않고 다양한 형태로 새로운 클래스를 개발할 수 있다
- 클래스의 다중 상속을 지원하지 않지만, 인터페이스로 다중 상속 효과를 간접적으로 얻을 수 있다

인터페이스 + 추상 클래스
- 구현 메서드 
    - 인터페이스 : 포함 불가
    - 추상 : 포함 가능
- 인스턴스 변수
    - 인터페이스 : 포함 불가
    - 추상 : 포함 가능
- 다중 상속
    - 인터페이스 : 가능
    - 추상 : 불가능
- 디폴트 메서드
    - 인터페이스 : 선언 가능
    - 추상 : 선언 불가능
- 생성자와 메인
    - 인터페이스 : 선언 불가능
    - 추상 : 선언 가능
- 상속에서의 부모
    - 인터페이스 : 인터페이스
    - 추상 : 인터페이스, 추상
- 접근 범위
    - 인터페이스 : 모든 멤버를 공개
    - 추상 : 추상 메서드를 최소한 자식에게 공개

인터페이스의 상속과 구현 클래스
인터페이스의 다형성
- 인터페이스도 업케스팅, 다운케스팅 쌉 가능함

중첩 클래스와 중첩 인터페이스
중첩 클래스는 외부 클래스를 상속할 필요없이 외부 클래스의 프라이베잇 멤버까지 사용할 수 있다

익명 클래스
- 이름없이 일회성 객체를 생성할 수 있도록 해주는 클래스이며, 인터페이스나 추상 클래스의 간단한 구현에 자주 사용됨

제네릭
- 정의와 종류
클래스 내부에서 사용할 데이터 타입을 외부에서 지정하는 기법
- 데이터 타입을 일반화하는 것을 의미
- 클래스나 메소드에서 사용할 내부 데이터 타입을 컴파일 시 미리 저장하는 방법
- 제네릭 사용시에 클래스나 메소드 내부에 사용되는 객체의 타입 안정성 향상
- 반환 값에 대한 타입 변환 및 타입 검사에 들어가는 비용 절감

오브젝트 특
- 자료형을 확실하게 보장해주지 않음

제네릭 인터페이스를 구현한다면 그 상태가 제대로 정해지고 나옴
