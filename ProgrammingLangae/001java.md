## 자바의 특징
### 언어적 특징
**ood** : 객체지향프로그래밍
    - 상속
    - 추상화

### 파일적 특징
- 컴파일 언어 : 바이너리 파일 전에 중간 파일이 나옴

**exe** : 컴파일 된 바이너리 파일
**바이트 파일** : 고급언어를 기계어로 바꾼 파일
    - 하지만 특정 하드웨어에 종속적임

그러나 자바는 크로스 플랫폼 지원으로 자바가 설치된 어느 컴퓨터이든 하드웨어가 무엇이든 다돌아간다 (이는 씨샵도 마찬가지다)

**중간파일** : 특정 가상기계가 읽을 수 있음 
    - 자바의 가상컴퓨터 jvm 
    - 씨샵의 가상컴퓨터 .net

---
## 자바 개발 과정
1. 소스 파일 작성 : 코드 작성
2. 컴파일 : 소스 파일을 바이트 코드로 생성
3. 로딩 : 클래스 로더가 바이트코드를 jvm에 로드
4. 링킹 : 검증, 준비, 해석 단계 거쳐 실행 준비
5. 초기화 : 클래스 변수 초기화 및 정적 블록 실행
6. 실행 : 인터프리터나 컴파일러가 바이트코드 실행

## 자바개발툴
### jvm
- 자바 소스코드로부터 만들어지는 자바 바이트 코드 실행기
- 하드웨어에 종속적임
- 컴파일된 바이트 코드는 어떤 jvm에서도 동작

### jre
- 컴파일된 자바 프로그램을 실행시킬 수 있는 자바 환경

### jdk
- 자바 프로그래밍시 필요한 컴파일러 등 도구 포함
- 보통 jdk설치시 jre도 설치됨
