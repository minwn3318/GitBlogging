# 쿠버네티스 아키텍처
컨테이너형 애플리케이션의 배포, 확장, 관리를 자동화하는 오픈소스 시스템

1. 마스터 노드
    - 스케줄러 : pod (container) 생성 시 위치 정하기
    - 컨트롤러 : 클러스터의 상태를 조절
    - api서버 : 유저에게 요청 받음
    - 저장소 : 클러스터 저장소

2. 워크 노드
    - 컨테이너 런타임 : 컨테이너를 실행하고 노드에서 컨테이너 이미지를 관리(실제 수행)
    - kubelet : 각 노드의 에이전트, pde에서 컨테이너가 확실하게 동작하도록 관리(서버와 연결 및 생성전달)
    - kube-proxy : 내부 네트워크 세션과 클러스터 바깥에서 pod로 통신할 수 있게 지원, 각 노드의 네트워크 규칙 유지 관리
