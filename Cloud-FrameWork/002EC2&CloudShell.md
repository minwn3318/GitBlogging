# EC2 인스턴스 생성

1. EC2 생성
- 인스턴스 생성

2. 이름 및 태그
- 이름 작성

3. 애플리케이션 및 OS 이미지
- 생성할 Instance의 이미지를 선택합니다 (보통 ubunto)

4. 인스턴스 유형 선택
- 원하는 인스턴스 선택 (보통 t3.micro)

5. 키 페어 로그인
- 새로운 키 페어를 생성하고 (보통은 기존 설정 유지), **절대** 로컬에 저장하기

6. 네트워크 설정
- VPC 설정 (보통 default vpc)
    - default는 테스트 용으로만 삿용 실전에서는 흠 그런디
- 퍼블릭 IP 자동 할당 설정 (보통은 활성화)
- 방화벽 설정 (보통은 기존 보안 그룹 선택)
    - 보안그룹도 보통 
- 인스턴스 시작 클릭으로 인스턴스 생성

7. 인스턴스 확인
- 검사 통과 확인
- 인스턴스 상태 (중지, 재부팅, 종료(삭제))

8. SSH 연결
- 인스턴스 연결의 SSH 연결 명령 복사
- CloudShell 검색하고 선택
- keypair 업로드
- pem key 권한 설정
```
chmode 400 "****.pem"
```
- ssh 연결 명령 붙여서 연결
```
ssh -i "****.pem" 이미지OS@ec2퍼블릭DNS이미지OS상세버전
```
