## 코드파이프라인 사용
### 사용 툴
s3
- 소스용 버킷
- 페이징용 버킷

code pipeline
- 배포용 파이프 라인

### 진행방법
1. 소스용 버킷
- 버킷 버전 활성화 체크
- 생성
- 2의 페이징용 버킷 다 사용 후, 페이징 버킷에 쓰일 인덱스문서와 오류문서의 압축 파일을 업로드

2. 페이징용 버킷
- ACL 활성화 체크
- 모든 퍼블릭 액세스 차단 해제 후, 퍼블릭 상태 가능 체크
- 생성
- 속성의 편집에서 정적 웹 사이트 호스팅 활성화
- 인덱스 문서와 오류 문서 지정
- 엔드포인트 활성 확인

3. 배포용 파이프 라인
- 파이프 라인생성
- 사용자 지정 파이프라인 빌드
- 파이프라인 이름 입력
- **소스 스테이징**
    - 소스 공급자 지정 (s3)
    - 소스 버킷명 지정
    - s3객체 키 지정 (소스 버킷 안에 있는 짚 파일)
- 빌드 스테이지와 테스트 스테이지는 건나감
- **배포 스테이지**
    - 배포 공급자 지정
    - 페이징 버킷 지정
    - 배포하기전 파일 압축
    - 표준 acl을 public-read로 지정
- 파이프라인 생성

### 내용 수정 방법
1. 파이프라인 비활성화 
2. 소스 파일 수정 후 제업로드 (버전 관리 확인 됨)
3. 파이프라인 활성화
